O Fim dos Conflitos de Horário: Grafos no Ensalamento

> status: published > module: graph_theory > complexity: medium

Todo início de ano letivo é a mesma história: coordenadores pedagógicos presos em uma sala com post-its coloridos, tentando encaixar 15 turmas, 20 professores e restrições infinitas. O ensalamento manual é um problema NP-difícil disfarçado de rotina.

Mas e se olharmos para a escola como um Grafo Matemático?

// 01. A MODELAGEM (NODES & EDGES)

A teoria dos grafos nos permite traduzir o caos escolar em uma estrutura lógica:

  • Vértices (Nós): Cada aula que precisa acontecer (ex: Mat-9A, Port-8B).
  • Arestas (Linhas): Um conflito. Se o Professor João dá aula de Mat-9A e Port-8B, existe uma linha conectando esses dois nós. Eles não podem acontecer ao mesmo tempo.
(MAT-9A) ------- (PORT-8B) | | | CONFLITO! | | | (HIST-9A) | \_________________/ [VISUALIZAÇÃO ABSTRATA DE CONFLITOS]

// 02. ALGORITMO DE COLORAÇÃO

Aqui entra a mágica: o problema do ensalamento é, matematicamente, um problema de Coloração de Vértices.

O objetivo é colorir cada nó de forma que dois nós conectados não tenham a mesma cor. Se traduzirmos "Cor" para "Horário" (ex: Cor Vermelha = Segunda 07:30), resolvemos a grade.

WARNING: O objetivo é encontrar o "Número Cromático" mínimo, ou seja, a menor quantidade de horários necessários para acomodar todas as aulas sem conflitos.

// 03. IMPLEMENTAÇÃO EM PYTHON

Não precisamos resolver isso no papel. Usando a biblioteca NetworkX, podemos automatizar a busca pela grade perfeita em milissegundos.

Abaixo, um protótipo do script que desenvolvi para simular essa alocação:

import networkx as nx
import matplotlib.pyplot as plt

# 1. Criando o Grafo Escolar
G = nx.Graph()

# 2. Definindo Aulas (Nós)
aulas = ["Matematica_9A", "Portugues_9A", "Matematica_8B", "Historia_9A"]
G.add_nodes_from(aulas)

# 3. Definindo Conflitos (Arestas)
# Ex: Mesma turma (9A) não pode ter Mat e Hist ao mesmo tempo
# Ex: Mesmo professor não pode estar em duas salas
conflitos = [
    ("Matematica_9A", "Historia_9A"), # Mesma Turma
    ("Matematica_9A", "Matematica_8B"), # Mesmo Professor
    ("Portugues_9A", "Historia_9A")
]
G.add_edges_from(conflitos)

# 4. Aplicando Algoritmo de Coloração (Greedy)
solucao = nx.coloring.greedy_color(G, strategy="largest_first")

# Output: {Aula: Horário_ID}
print("Alocação Otimizada:", solucao)
# Resultado: {'Matematica_9A': 0, 'Historia_9A': 1, ...}

// 04. RESULTADO NA PRÁTICA

Ao aplicar essa lógica, transformamos dias de trabalho manual em uma execução de script. Além de evitar erros humanos (dois professores na mesma sala), o algoritmo pode ser ajustado para otimizar "janelas" e preferências dos docentes.

Isso não é substituir o professor pela máquina, é usar a máquina para que o professor tenha tempo de ser professor.
